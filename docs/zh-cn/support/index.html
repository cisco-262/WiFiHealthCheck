WiFi Health Check

支持

这里提供产品支持、排障方向，以及 iOS WiFi 信息限制说明。

联系方式

WiFi Health Check 支持请联系 support@opshome.run。

请提供应用版本、iOS 版本、设备型号和问题描述。不要发送 WiFi 密码、路由器管理密码或私有凭据。

常见排障说明

  • 如果 SSID、BSSID 或 WiFi 安全类型不可用，请确认你使用真实 iPhone 且已连接 WiFi，不是模拟器或蜂窝网络。
  • 如果需要读取当前 WiFi 详情，请允许位置权限。iOS 对 WiFi 标识符和当前网络信息有此要求。
  • 网关 TCP 端口关闭不等于互联网断开。公网网站连通性才是主要互联网信号。
  • LAN TCP 稳定性测试需要填写路由器、NAS、打印机、AP 或服务器实际使用的端口。
  • 漫游测试时请保持应用在前台，并在 WiFi 节点之间移动。
  • 普通 App Store 应用无法可靠获取 RSSI、频道、频段、信道宽度或 PHY 速率。

什么时候使用 OpsHome NOC

如果你需要持续监控、告警、Docker Probe、Synology、Proxmox、Linux 主机、私有服务检测或历史基础设施看板，请使用 OpsHome NOC。
