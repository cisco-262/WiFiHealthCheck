WiFi Health Check

Privacy Policy

WiFi Health Check is designed as a local foreground diagnostic app. Diagnostic reports are stored on the device unless you choose to share them.

1. Overview

WiFi Health Check helps users run local WiFi and LAN diagnostics, including public website reachability, current WiFi context, gateway information, custom TCP stability tests, and WiFi roaming reports.

2. Data Processed on Device

The app may process the following information locally on your iPhone:

  • Current WiFi SSID, BSSID, and security type when iOS provides them
  • Local IP address and estimated gateway address
  • Public website connectivity test results
  • Custom LAN target IP address and TCP port entered by you
  • Diagnostic history, stability samples, and roaming samples
  • Language preference and saved local report history

3. Location Permission

iOS requires Location permission for apps to access current WiFi information such as SSID, BSSID, and security type. WiFi Health Check uses this permission only to display and record current-network context for diagnostics. The app does not need your physical location history.

4. Cloud and Account

WiFi Health Check does not require account registration and does not upload diagnostic history to a cloud service. Reports leave your device only when you explicitly share them through iOS.

5. Data We Do Not Collect

  • No router passwords
  • No WiFi passwords
  • No RSSI, channel, frequency band, or PHY rate because iOS does not provide those values to ordinary App Store apps
  • No packet capture data
  • No browsing history
  • No account identity for app registration

6. Support

For privacy-related questions, contact OpsHome support. Do not send WiFi passwords, router admin passwords, or private network credentials.
