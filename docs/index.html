How WiFi Health Check works

Open the app on WiFi

Use a real iPhone connected to WiFi. iOS may require location permission to expose SSID, BSSID, and security type.

Run a focused test

Choose one-tap diagnosis, LAN TCP stability, or roaming test depending on the problem you are investigating.

Review local results

Understand public-site reachability, WiFi security type, gateway context, custom-port stability, and roaming path changes.

Share only when needed

Reports stay on your device unless you explicitly share them from iOS.

Know whether the problem is internet, LAN, or roaming

A home or office WiFi issue can come from broadband, DNS, router WAN, local device reachability, weak signal, or a sticky client attached to the wrong access point.

WiFi Health Check separates those signals into small reports you can compare across locations.

Frequently Asked Questions

Why does the app ask for Location permission?

iOS gates WiFi SSID, BSSID, and current-network details behind Location permission and WiFi information entitlement support. The app uses that access to display the connected WiFi name, node identifier, and security type when available.

Does a failed gateway TCP probe mean my internet is down?

No. Routers can use custom ports or close common ports. WiFi Health Check uses public website reachability as the main internet connectivity signal and keeps gateway TCP status as supporting context.

What does BSSID unavailable mean?

It means iOS did not provide the WiFi node identifier to the app. This can happen on simulator, cellular, missing permission, entitlement limitations, or system privacy restrictions.

Can the app show WiFi signal strength or channel?

iOS does not provide reliable RSSI, channel, frequency band, channel width, or PHY rate to ordinary App Store apps. WiFi Health Check shows the WiFi details iOS allows, then uses connectivity, DNS, TCP, latency, timeout, and roaming behavior to diagnose real network experience.

Can I test a custom gateway or NAS port?

Yes. The LAN TCP stability test lets you enter an IP address and TCP port for repeated connectivity sampling.

Does this replace long-term monitoring?

No. It is a foreground local diagnostic tool. Use OpsHome NOC if you need continuous uptime, Docker, Synology, Proxmox, Linux host, or private infrastructure monitoring.

Need continuous infrastructure monitoring?

WiFi Health Check is for local foreground diagnosis. OpsHome NOC is built for ongoing monitoring of services, Docker, Synology, Proxmox, Linux hosts, private endpoints, alerts, and history.
