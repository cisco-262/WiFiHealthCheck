WiFi Health Check

Support

Use this page for product support, troubleshooting direction, and notes about iOS WiFi information limits.

Contact

For WiFi Health Check support, contact OpsHome at support@opshome.run.

Please include your app version, iOS version, device model, and a short description of the issue. Do not send WiFi passwords, router admin passwords, or private credentials.

Common troubleshooting notes

  • If SSID, BSSID, or WiFi security type is unavailable, confirm you are using a real iPhone on WiFi, not Simulator or cellular.
  • Allow Location permission if you want the app to read current WiFi details. iOS requires this for WiFi identifiers and related current-network information.
  • A closed gateway TCP port does not automatically mean the internet is down. Use public website connectivity as the primary internet signal.
  • For LAN TCP stability, enter the correct custom port used by your router, NAS, printer, AP, or server.
  • For roaming tests, keep the app in the foreground while walking between WiFi nodes.
  • iOS does not expose reliable RSSI, channel, frequency band, channel width, or PHY rate to ordinary App Store apps. Use the app's DNS, TCP, latency, timeout, and roaming results to understand connection quality.

When to use OpsHome NOC

If you need continuous monitoring, alerts, Docker Probe, Synology, Proxmox, Linux host monitoring, private service checks, or historical infrastructure dashboards, use OpsHome NOC.
